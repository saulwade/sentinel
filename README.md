# Sentinel

**DevTools for AI agents.** Watch them. Rewind them. Fork reality. Ship safer agents.

> Your agent just did something weird in production. Today you read logs. With Sentinel, you scrub the timeline, edit the past, and replay the future.

Six reasoning tasks. One model: **Claude Opus 4.7.**

Built for the [Built with Opus 4.7 Hackathon](https://cerebralvalley.ai/events/~/e/built-with-4-7-hackathon) by [@saulwade](https://github.com/saulwade).

---

## What it does

Sentinel is a **debugger for autonomous AI agents** &mdash; like Chrome DevTools, but for agents that read emails, query databases, and take actions on your behalf.

It sits between your agent and its tools. Every action is intercepted, evaluated by Opus in real-time, and streamed to a visual timeline you can scrub, edit, and fork.

### The demo scenario

A corporate assistant agent is asked to *"summarize my unread emails."* One email contains a hidden prompt injection instructing the agent to exfiltrate customer data. Sentinel catches the attack chain before any data leaves the system.

## The five tabs

### 1. Live View
Real-time action stream. Each tool call gets a Pre-cog verdict:
- **ALLOW** (green) &mdash; safe, executed
- **PAUSE** (amber) &mdash; suspicious, human decides
- **BLOCK** (red) &mdash; dangerous, stopped before execution

Opus extended thinking streams in purple so you see *why* it decided.

### 2. Timeline (Time-Travel)
Every agent step is an event stored in SQLite. Drag the scrubber to any point and see the exact world state: which emails were read, what data was queried, what was sent. Edit the state and press **Replay** to fork reality.

### 3. Fork View
Side-by-side comparison of two branches:
- **Original run** &mdash; what happened (attack blocked)
- **Forked run** &mdash; alternate reality (phishing email removed, agent runs clean)

Opus narrates the difference at the bottom in purple.

### 4. Pre-flight Simulator
Before deploying, run your agent through 30 simulated days of synthetic scenarios &mdash; normal emails, edge cases, and adversarial attacks &mdash; all generated by Opus. Get a safety grade (A+ to F) with failure drill-down.

### 5. Red Team
Opus generates novel prompt injection attacks tailored to your agent's tools and system prompt. Each is tested against Pre-cog. Bypasses get auto-generated defense policies.

## How Opus 4.7 is used

Six distinct reasoning tasks, none of which work with smaller models:

| Feature | What Opus does | Reasoning type |
|---|---|---|
| Pre-cog | Simulates causal chain of proposed tool calls | Long-horizon causal reasoning |
| Time-Travel | Re-simulates agent under edited context | Counterfactual + agentic reasoning |
| Fork Narrator | Narrates the branch-not-taken | Counterfactual narration |
| Pre-flight | Generates plausible synthetic scenarios | World-modeling / simulation |
| Red Team | Produces tailored adversarial attacks | Theory-of-mind of target agent |
| Policy Synthesis | Turns bypasses into defense rules | Spec synthesis |

All reasoning uses **extended thinking** with streaming, visible in the UI as purple text.

## Architecture

```
                    localhost:3000                    localhost:3001
               ┌─────────────────────┐          ┌─────────────────────────┐
               │   Next.js 16 App    │          │      Hono Engine        │
               │                     │   SSE    │                         │
               │  Live View ─────────┼──────────┤─► Agent Runner          │
               │  Timeline ──────────┼──────────┤─► Event Store (SQLite)  │
               │  Fork View ─────────┼──────────┤─► Time-Travel Engine    │
               │  Pre-flight ────────┼──────────┤─► Pre-flight Simulator  │
               │  Red Team ──────────┼──────────┤─► Red Team Attacker     │
               │                     │          │                         │
               └─────────────────────┘          │  ┌───────────────────┐  │
                                                │  │  Tool Interceptor │  │
                                                │  │  ┌─────────────┐  │  │
                                                │  │  │  Pre-cog    │──┼──┼──► Opus 4.7
                                                │  │  │  (verify)   │  │  │   (extended thinking)
                                                │  │  └─────────────┘  │  │
                                                │  └───────────────────┘  │
                                                │                         │
                                                │  MCP Server (stdio)     │
                                                │  ► Claude Code compat   │
                                                └─────────────────────────┘
```

**Stack:** TypeScript end-to-end. Next.js 16 + React 19 + Tailwind 4 (web). Hono + SQLite + Drizzle ORM (engine). MCP SDK. Anthropic SDK with streaming.

**Event sourcing:** every agent interaction is an immutable event row in SQLite. World state at any point = replay events 0..N. Forks create new runs with `parentRunId` set. No mutation, full auditability.

## Quick start

```bash
git clone https://github.com/saulwade/sentinel.git
cd sentinel
pnpm install
cp apps/engine/.env.example apps/engine/.env
# Add your ANTHROPIC_API_KEY to apps/engine/.env
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000). Press **Run agent** (or hit `R`).

**Requirements:** Node 22+, pnpm 9+, Anthropic API key with Opus access.

### Keyboard shortcuts

| Key | Action |
|---|---|
| `1`&ndash;`5` | Switch tabs |
| `R` | Run agent |

## MCP integration (Claude Code)

Sentinel is an MCP server. Add it to Claude Code:

```json
{
  "mcpServers": {
    "sentinel": {
      "command": "pnpm",
      "args": ["-F", "@sentinel/engine", "mcp"],
      "cwd": "/path/to/sentinel"
    }
  }
}
```

Then in Claude Code:

```
> Start a Sentinel run and tell me what Pre-cog blocked
> Show me the world state at event #5
> List the agent's available tools
```

MCP tools: `sentinel_start_run`, `sentinel_get_events`, `sentinel_snapshot`, `sentinel_list_agent_tools`.

## Why Sentinel exists

AI agents are shipping to production every week. When they fail, the answer today is "read logs and guess." Sentinel gives agent developers a real debugger:

- **See** every action in real-time with causal reasoning
- **Pause** suspicious actions before they execute
- **Rewind** to any point in the agent's history
- **Edit** the past and replay alternate futures
- **Test** against hundreds of synthetic scenarios before deploying
- **Discover** novel attacks before attackers do

This is a debugging primitive that did not exist before models could reason about counterfactuals.
